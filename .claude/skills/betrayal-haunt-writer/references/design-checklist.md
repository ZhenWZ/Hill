# 设计准则与校验协议 / Design Principles & Validation Protocol

## A. 可玩性设计准则 / Playability Principles

1. **有限收束**：必须存在一个机制保证游戏在闹鬼后约 8–12 个整轮内结束
   （Turn/Damage Track 倒计时、怪物逐轮变强、房间逐轮坍塌等）。"双方都赢不了"的静坐僵局 = 设计失败。
2. **互斥胜利**：双方胜利条件不可能同时满足；判定时点明确（"立即获胜"）。
   奸徒条件通常附带 `or else all heroes are dead / 或所有英雄全部死亡`。
3. **任务链可达**：英雄任务链总步骤 × 平均检定成功率，应使 3–5 名英雄在节奏钟耗尽前
   有真实机会完成（粗算：步骤数 ≤ 轮数上限 × 英雄数 × 0.4）。
4. **开局生存空间**："Right Now" 不得直接杀死或锁死任何一方；怪物初始位置须与英雄保持距离
   （官方惯例："同层至少 5 格远，若无则尽量远"）。
5. **奸徒有事可做**：奸徒每回合应有有意义的决策（推进自己的进度条/操控怪物/拦截英雄），
   而不是纯等待英雄失败。
6. **关键组件兜底**：胜利条件依赖的每张卡牌/每个房间都要有"不在场上时"的兜底规则（官方句式见 haunt-format.md）。
7. **怪物量级**：杂兵 Speed 2–4 / Might 2–4；精英 4–6；单体 BOSS Might 可到 8（官方木乃伊 Speed 3 Might 8 Sanity 5）。
   多怪物时用编号圆 token 并写明数量上限（受实体 token 数限制）。
8. **人数适应**：写明 3–6 人差异（如 token 数 = 玩家数；官方惯用 "equal to the number of players"）。
9. **隐藏奸徒 (None at first)** 剧本必须额外写明：奸徒产生的触发条件、产生前双方读哪一册、穿帮防护。
10. **双语一致**：中英文本对译后规则数值、房间名、判定语句完全一致；以英文为规则基准文本。

## B. 自检清单 / Pre-validation Self-check（写完两册后逐项过）

- [ ] 两册编号、标题、触发组合、奸徒选择语句一致
- [ ] 每个检定写明属性与目标值；每个 token 写明形状与放置位置
- [ ] 英雄册 "What You Know About the Bad Guys" ↔ 奸徒册实际目标：一致且不剧透
- [ ] 怪物属性条齐全；攻击/免疫规则无自相矛盾
- [ ] 所有引用组件均在 components.md 清单内或为泛指
- [ ] 节奏钟存在且两册描述一致
- [ ] 中英内容逐节对应、数值一致

## C. Subagent 校验协议 / Validator Protocol

三个校验员**并行**派发（Agent 工具，subagent_type: `general-purpose`）。每个 prompt 必须包含：
待校验文件的绝对路径、本文件与 game-rules.md / components.md 的绝对路径、下方对应的职责说明，
并以如下格式结尾要求结构化输出：

```
最后以一行 "VERDICT: PASS" 或 "VERDICT: FAIL" 结束；若 FAIL，在其前列出编号问题，
每条注明 [文件名 § 小节] 问题描述 + 建议修法。轻微文字问题列为 NOTE，不构成 FAIL。
```

### Validator 1 — 英雄剧本校验员 Hero-book validator

只读 `secrets-of-survival.md` + 规则/组件参考。检查：

- 所有机制符合 game-rules.md；所有组件在 components.md 内或为合法泛指
- 任务链每一步：地点存在、有兜底、检定格式正确、token 来源明确
- 指令无歧义（一个新玩家照字面执行不会卡住）
- "每回合一步"之类的限速语句存在
- 中英一致

### Validator 2 — 奸徒剧本校验员 Traitor-book validator

只读 `traitors-tome.md` + 规则/组件参考。检查同上，外加：

- 怪物属性量级符合本文件 A.7；怪物回合规则完整（移动、攻击、特殊行为）
- 奸徒自身的行动空间（A.5）
- "Right Now" 的放置规则可执行（包括地图很小/很大的极端情况）

### Validator 3 — 胜利条件交叉校验员 Win-condition validator

读两册 + `design-notes.md` + 本文件。检查：

- 双方胜利条件互斥、判定时点明确
- 无僵局：按 A.1 验证有限收束；模拟"双方都摆烂"的极端情形仍有一方会赢
- 信息对称性穿帮：两册对同一可见事实无矛盾；英雄册没有泄露奸徒暗步骤，反之亦然
- 粗略平衡（A.3 公式 + 怪物压制力定性评估）：估计双方胜率，超出 35%–65% 即 FAIL 并给出调参建议
- 3 人局与 6 人局都成立

### 修订循环 / Revision Loop

FAIL → 修复全部所列问题 → 仅重派失败的校验员 → 最多 2 轮。
仍未通过：交付时在 design-notes.md「已知问题」节如实记录。
